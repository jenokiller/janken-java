# janken-java

## 概要
- このプロジェクトは、テスト駆動開発でじゃんけんプログラムを作る場合の完成サンプルです。
- テスト駆動開発は、テストを先に書いて、そのテストを通すような実装(プログラム)を書く開発手法です。
- 詳しい内容は、テスト駆動開発で調べてください。

## 動かす前提条件
- Gitがインストールされている
- 統合開発環境がインストールされている(STS、VSCode、Intellij)

## 動かし方

- コマンドプロンプト or ターミナルを開く

- `cd`コマンドで、自由に使ってよいディレクトリに移動する

- 下記のコマンドを実行し、プロジェクトを複製(クローン)&ディレクトリに移動する
  ```
  # git clone https://github.com/jenokiller/janken-java.git
  # cd janken-java
  ```
  
- 下記のコマンドを実行し、テストを実行する
  ```
  # gradlew test
  ```
  > BUILD SUCCESSFULの文字が出れば、成功
  

## テスト駆動開発でじゃんけんプログラムを作る(1テストケース分)
### はじめに
- Janken.javaの5~12行目を`return "";`に置き換える
- JankenApplicationTestsの21~44行目を削除する
- 下記のコマンドを実行し、テストを実行する
  ```
  # gradlew test
  ```
  > BUILD SUCCESSFULの文字が出れば、準備完了

### STEP1(テストを書く)
- JankenApplicationTestsに下記のソースを追加する
  ```
	@Test
	public void P1がグーP2がチョキの時P1が勝つこと() {
		assertThat(janken.play(ROCK, SCISSORS), equalTo("p1"));
	}
  ```

### STEP2(テストを実行する)
- 下記のコマンドを実行し、テストを実行する
  ```
  # gradlew test
  ```
  > BUILD FAILEDの文字が出れば、正しい  
  > まだ、実装を書いていないのでテストは必ず失敗する
  
### STEP3(実装を書く)
- Janken.javaにSTEP1で書いたテストが通る必要最小限の実装を書く
- どう書くかは、考えてみてください
- ここでのポイントは、先の処理を想像して無駄な実装をしないこと
- p1の文字列が返却されれば、テストは通るので`return "";`の部分を`return "○○";`にする

### STEP4(再度、テストを実行する)
- 下記のコマンドを実行し、テストを実行する
  ```
  # gradlew test
  ```
  > BUILD SUCCESSFULの文字が出れば、1テストケースが完成
  
### 最終的に
STEP1に戻って新しいテストを作成し、STEP4までを繰り返すことでじゃんけんプログラムが徐々にできます。  
最終的にじゃんけんプログラムができている頃には、必要なテストが出来上がった状態で完成します。  
テスト駆動開発は、テストを積み上げてコードの品質を確保する画期的な開発手法です。
